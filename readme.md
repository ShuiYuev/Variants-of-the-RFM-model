# 用户价值分层用以精准营销 — 项目流程说明

## 一、业务困境

当前面临三个核心问题：

1. **老用户复购持续下降** — 曾经活跃的高价值用户正在流失，需要提前预警
2. **潜水用户被误判** — 很多新用户频繁浏览（高意向），但购买很少或零购买，传统RFM模型只看R（最近购买时间）、F（购买频次）、M（消费金额），会将这类用户直接判为"低价值用户"，后续不对他们做精细化运营，导致潜在用户白白流失
3. **粗放式运营效率低** — 传统三维RFM分出的群体颗粒度太粗，无法区分"真正没价值的用户"和"有兴趣但还没转化的用户"

## 二、解决方案核心思路

### 为什么传统RFM不够用？

传统RFM只有三个维度：

| 维度 | 含义 | 问题 |
|------|------|------|
| R (Recency) | 最近一次购买距今多久 | 只能看"买了没"，看不出"看了没" |
| F (Frequency) | 购买了多少次 | 只看结果不看过程 |
| M (Monetary) | 消费了多少钱 | 同上 |

一个典型场景：用户A过去90天浏览了300分钟、40个页面，但只买了1次。传统RFM看F=1，M=100，直接判为"低价值"。实际上这个人兴趣很强（意向度很高），只是转化有障碍（摩擦系数大），给一张优惠券就很可能变成高价值用户。

### 引入两个新指标

#### 1. 意向度 (Intent Score)
```
意向度 = 浏览时长 × 0.5 + 浏览页数 × 0.5
```
- **含义**：用户对平台内容的兴趣强度和浏览深度
- **为什么要加**：在购买行为发生之前，浏览行为已经能反映用户的兴趣倾向。高意向 + 低购买 = 有潜力待挖掘

#### 2. 转化摩擦系数 (Conversion Friction)
```
转化摩擦系数 = 购买次数 / (1 + 浏览页数)
```
- **含义**：用户从浏览到购买的转化效率。值越小说明浏览了很多但买得少——"摩擦力"大，需要运营干预
- **为什么要加**：识别"浏览多买得少"的用户。同是高意向用户，摩擦系数低的需要降低购买门槛（优惠券），摩擦系数高的说明转化路径顺畅，可以推高客单价

### 增强RFM = R + F + M + 意向度 + 转化摩擦系数

用五个维度做K-Means聚类，能将用户分成更精细的群体，尤其能把"潜水/摩擦型用户"单独识别出来。

## 三、项目流程

```
数据加载 → EDA与可视化 → 特征工程(计算两个新指标) → 数据标准化
→ 肘部法则+轮廓系数确定最佳K值 → K-Means五维聚类
→ 群体画像分析+自动打标签 → 对比传统RFM的误判情况
→ 制定差异化运营策略 → 导出结果
```

### 步骤详解

#### Step 1: 数据加载与探索
- 输入：`user_personalized_features.xlsx`（1000个用户，12个字段）
- 核心字段：距上次登录天数、购买频次、总消费金额、浏览时长、浏览页数等
- 检查缺失值、分布特征

#### Step 2: 特征工程（核心创新点）
- 根据业务公式计算意向度和转化摩擦系数
- 分析"潜水用户"（高意向 + 低转化效率或零购买）的规模和特征

#### Step 3: 构建聚类特征矩阵
- R取负值：使方向与其他指标一致（距上次登录越近，值越大）
- 五个特征做相关性分析，确认没有高度冗余
- Z-Score标准化

#### Step 4: 确定最佳K值
- **肘部法则**：观察簇内平方和（Inertia）随K的变化，找到"拐点"
- **轮廓系数**：衡量聚类结果中样本的凝聚度和分离度，取轮廓系数最大的K
- 两个方法联合判断

#### Step 5: K-Means聚类
- 用最佳K值对五维特征进行聚类
- 还原聚类中心到原始量纲，便于业务理解
- 雷达图 + 箱线图 + 散点图 + PCA降维图，多角度可视化

#### Step 6: 群体画像与标签
- 每个群体在各维度上排名
- 自动规则引擎给每个群体打业务标签，如"潜水/摩擦型用户"、"高价值流失预警"等
- 核心标签说明：
  - **潜水/摩擦型用户**：意向度高但购买少，传统RFM中的"隐形金矿"
  - **高意向待转化(低摩擦)**：意向度高、转化路径顺畅但还没怎么买——最容易转化的群体
  - **高价值流失预警**：曾经高消费但近期不活跃，需要召回
  - **高价值核心用户**：R、F、M都优秀，需要维护和升级

#### Step 7: 与传统RFM对比（验证模型价值）
- 用相同的K值，仅用R/F/M三维再做一次聚类
- 交叉表对比两个模型的群体分配
- 重点看："传统RFM判为低价值的用户，在增强模型中去了哪里"
- 量化指标：
  - 传统RFM低价值群体中，有多少比例是高意向用户
  - 高意向低购买用户在两个模型中的分布差异
  - 零购买但高意向的用户是否被单独识别出来

#### Step 8: 运营策略矩阵
- 为每个群体制定差异化运营策略
- 例：潜水用户 → 首单优惠券 + 个性化推荐 + 购物车召回
- 例：流失预警 → 客服回访 + 大力度回归券 + 情感化召回

#### Step 9: 结果导出
- 输出 `user_segmentation_enhanced_rfm.xlsx`，包含用户ID、群体标签及所有特征值

## 四、关键指标说明

| 指标 | 公式 | 业务含义 | 越高代表 |
|------|------|----------|----------|
| R (取负后) | -Last_Login_Days_Ago | 用户活跃近度 | 近期越活跃 |
| F | Purchase_Frequency | 购买频次 | 越忠诚 |
| M | Total_Spending | 消费金额 | 越有消费力 |
| 意向度 | 时长×0.5 + 页数×0.5 | 浏览兴趣强度 | 越感兴趣 |
| 转化摩擦系数 | 购买次数/(1+浏览页数) | 浏览→购买转化效率 | 转化越顺畅 |

## 五、预期业务效果

| 目标 | 具体手段 | 预期提升 |
|------|----------|----------|
| 潜水用户转化 | 首单优惠券 + 个性化推荐 + 购物车召回 | 转化率提升10-15% |
| 老用户流失减缓 | 摩擦系数趋势监控 + 提前干预 | 流失率下降5-8% |
| 营销ROI提升 | 减少对真正低价值用户的无效触达 | 触达成本下降20%+ |
| 运营效率 | 从粗放式大群运营变为精准细分运营 | 人均产出提升 |

## 六、后续迭代方向

1. **时间维度**：定期（每周）更新聚类，监控用户在群体间的迁移路径
2. **A/B测试**：对"潜水/摩擦型"用户执行转化策略，量化效果
3. **实时化**：将意向度和摩擦系数纳入实时特征工程，支持在线推荐
4. **品类细化**：按商品品类分别建模，得到品类级别的用户分层
